SmartAdServerViewDelegate Protocol Reference

Declared in SmartAdServerViewDelegate.h

Overview

The delegate of a SmartAdServerView object must adopt the SmartAdServerViewDelegate protocol.

Many methods of SmartAdServerViewDelegate return the ad view sent by the message. The protocol methods allow the delegate to be aware of the ad-related events. You can use it to handle your app’s or the ad’s behavior like adapting your viewController’s view size depending on the ad being displayed or not.

sasView:shouldHandleUrl:

Asks the delegate whether to execute ad action

- (BOOL)sasView:(SmartAdServerView *)adView shouldHandleUrl:(NSURL *)URL

Parameters

URL

The url that will be called

Return Value

whether the Smart AdServer SDK should handle the URL

Discussion

Asks the delegate whether to execute ad action

Implement this method if you want to process some urls yourself.

Returning NO means that the URL won’t be processed by the SDK.

Please note that a click will be counted, even if you return “NO” (you are supposed to handle the URL in this case).

sasViewDidDisappear:

Tells the delegate that the SmartAdServerView has been dismissed

- (void)sasViewDidDisappear:(SmartAdServerView *)adView

Parameters

adView

The ad view informing the delegate that it was dismissed

Discussion

Tells the delegate that the SmartAdServerView has been dismissed

This can happen :

  • if the user taps the “Skip” button
  • if the ad’s duration elapsed
  • if the ad has been clicked
  • if the ad creative decided to close itself
  • if your application decided to dismiss it by calling [SmartAdServerView dismiss]

You should not call the adView in this method, except if you want to release it (set your property and the ad’s delegate to nil then)

  • display a custom SmartAdServerAd : see [SmartAdServerView displayThisAd:]
  • refresh the ad view : see [SmartAdServerView refresh]

  • dismiss the ad view

    [adView dismiss];

if it’s unlimited or remove it
